# 监控中心使用说明

## 1. 目录结构

```text
E:\tools\OB\Obsidian\Trade\notes\monitor
  config/
    sell-monitor-config.md
    sell-watchlist.md
    entry-watchlist.md
  sell/
  entry/
  launchers/
  docs/
```

## 2. 配置文件

- 主配置：`config/sell-monitor-config.md`
- 卖出名单：`config/sell-watchlist.md`
- 开仓名单：`config/entry-watchlist.md`

说明：

- 卖出监控只读取 `sell-watchlist.md`
- 开仓检查只读取 `entry-watchlist.md`
- 两份名单互不影响

## 3. watchlist 写法

```json
{
  "symbols": [
    {
      "symbol": "002241",
      "name": "歌尔股份"
    },
    "601888"
  ]
}
```

说明：

- `name` 可选
- 配了 `name` 时，表格里的“股票代码”列优先显示中文名

## 4. 常用启动器

启动器目录：

```text

```

常用文件：

- `启动开仓检查.bat`
- `启动开仓检查_每小时循环.bat`

如果你用 Obsidian 的 `Shell commands` 插件，Shell 请选择 `PowerShell`。

命令示例：

```powershell

```

```powershell
& "E:\tools\OB\Obsidian\Trade\notes\monitor\launchers\启动开仓检查.bat"
```

## 6. 输出位置

- 卖出监控结果：`monitor/sell/`
- 开仓检查结果：`monitor/entry/`
- 配置文件：`monitor/config/`
- 文档说明：`monitor/docs/`

重点汇总文件：

- 卖出：`sell/当日应卖出.md`
- 开仓：`entry/当日可开仓候选.md`

## 7. Telegram 提醒

需要配置：

```dotenv
SELL_MONITOR_TELEGRAM_BOT_TOKEN=your-bot-token
SELL_MONITOR_TELEGRAM_CHAT_ID=your-chat-id
SELL_MONITOR_TELEGRAM_SUBJECT_PREFIX=[SellMonitor]
```

触发条件：

- 卖出监控：动作 `reduce / stop_loss / exit_all`，且分数 `>= 5`
- 开仓检查：`allow_entry`

## 8. 开仓结果怎么解读

开仓检查现在有三条路线：

- `标准开仓`
- `T仓回补`
- `禁止回补`

当前统一放行线：

- 只要分数 `>= 5`
- 且盈亏比 `>= 1.5:1`
- 同时止损、第一止盈位明确
- 且没有硬阻断项

补充说明：

- `T仓回补` 只适用于 `pullback_buy`、`order_block_buy`
- `breakout_buy` 只用于标准开仓
