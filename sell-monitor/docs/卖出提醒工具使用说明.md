# 卖出提醒工具使用说明

本文说明如何把 A 股加入 watchlist、如何配置 Obsidian 输出路径，以及加入后如何启动卖出监控。

## 0. 配置文件与 Obsidian 输出路径

工具通过环境变量读取配置，默认先加载项目根目录下的 `.env` 文件：

```text
C:\Users\admin\Documents\New project\sell-monitor\.env
```

当前 Obsidian 监控文档输出目录已经做成可配置项，默认值是：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor
```

对应配置项：

- `SELL_MONITOR_OBSIDIAN_MONITOR_ENABLED`：是否启用写入 Obsidian，默认 `true`
- `SELL_MONITOR_OBSIDIAN_MONITOR_DIR`：Obsidian 监控文档输出目录
- `SELL_MONITOR_ENV_FILE`：可选，自定义 `.env` 文件路径

最常用的做法是直接编辑项目根目录下的 `.env`：

```dotenv
SELL_MONITOR_OBSIDIAN_MONITOR_ENABLED=true
SELL_MONITOR_OBSIDIAN_MONITOR_DIR=E:\tools\OB\Obsidian\Trade\notes\monitor
```

如果你要改到别的 Obsidian 库路径，只需要修改这一项：

```dotenv
SELL_MONITOR_OBSIDIAN_MONITOR_DIR=D:\Obsidian\Trade\notes\monitor
```

如果不想使用默认 `.env`，也可以在运行前指定另一份配置文件：

```powershell
$env:SELL_MONITOR_ENV_FILE="D:\configs\sell-monitor.env"
.\scripts\run_akshare.ps1 -Symbol 002241
```

关闭当前 PowerShell 窗口后，这个临时环境变量会失效；如果需要长期使用，写入系统环境变量或直接维护项目根目录下的 `.env` 更直接。

## 1. 推荐方式：运行一次单股检测，自动加入 watchlist

进入项目目录：

```powershell
Set-Location "C:\Users\admin\Documents\New project\sell-monitor"
```

运行单只股票检测：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241
```

执行后工具会自动做一件事：

- 如果 `002241` 不在 watchlist 中，会自动写入 `examples\watchlist.json`。

后续再运行全量监控时，这只股票会被一起监控。

## 2. 手动方式：直接编辑 watchlist

watchlist 文件路径：

```text
C:\Users\admin\Documents\New project\sell-monitor\examples\watchlist.json
```

格式示例 1：只写股票代码：

```json
{
  "symbols": [
    "002241",
    "002739",
    "601012"
  ]
}
```

格式示例 2：给股票设置自定义显示名称：

```json
{
  "symbols": [
    {
      "symbol": "002241",
      "name": "歌尔股份"
    },
    {
      "symbol": "300015",
      "name": "爱尔眼科"
    },
    "300014"
  ]
}
```

显示规则：

- 如果 watchlist 中给该股票设置了 `name`，监控结果中的“股票代码”一栏优先显示这个名称。
- 如果没有设置 `name`，则显示股票代码本身。
- 旧格式仍然兼容，直接写 6 位股票代码即可。

说明：

- 文档表格列名仍然写作“股票代码”，但显示内容会优先使用你在 watchlist 中配置的股票名称。

股票代码使用 6 位 A 股代码。深市、创业板不需要加 `SZ`，沪市、科创板不需要加 `SH`。

## 3. 启动监控

单次监控所有 watchlist 股票：

```powershell
.\scripts\run_akshare.ps1
```

单次监控一只股票：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241
```

每 1 小时自动监控所有 watchlist 股票：

```powershell
.\scripts\run_akshare.ps1 -Loop -IntervalSeconds 3600
```

每 1 小时自动监控一只股票：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241 -Loop -IntervalSeconds 3600
```

循环监控时保持 PowerShell 窗口打开。关闭窗口后，监控会停止。

实时检测默认只在 A 股常规交易时段运行：

- 交易日 `09:30-11:30`
- 交易日 `13:00-15:00`

非交易时段会打印“非A股交易时段，跳过本轮实时检测”，不会写入监控结果。若只是为了调试，需要在非交易时段强制运行一次，可以加：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241 -IgnoreTradingHours
```

## 4. 查看监控结果

监控结果会同时打印到终端，并写入 Obsidian。若未改配置，默认写入：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor
```

如果你在 `.env` 中修改了 `SELL_MONITOR_OBSIDIAN_MONITOR_DIR`，则会写入你指定的新目录。

每只股票一篇文档：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\002241.md
```

最新监控结果会插入到文档最上方，方便打开后直接看最新结论。

每一轮定时检测都会写入一条表格记录，即使没有卖出信号也会记录“未触发卖出信号”或数据获取提示。这样可以确认工具确实按时运行过。

股票监控文档中的表格字段包括：

- `检测时间`
- `股票代码`
- `结论`
- `动作`
- `分数`
- `原因/提示`
- `下一步`
- `取消条件`

如果触发卖出类信号，表格中的检测时间会标红：

- `reduce`：减仓提醒。
- `stop_loss`：止损提醒。
- `exit_all`：清仓提醒。

当日触发卖出信号的股票会额外汇总到：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\当日触发.md
```

该文档只记录卖出类信号，表格中会列出触发时间、股票代码、卖出动作、分数、触发原因和建议。

常见结论含义：

- `hold`：继续持有，暂不触发卖出提醒。
- `reduce`：减仓提醒。当前规则中，3 分以上会提示减仓。
- `exit_all`：清仓提醒。当前规则中，5 分以上会提示清仓。

降低误报后的清仓规则：

- 第三根危险上影线不再单独触发清仓，只作为高风险减仓信号。
- 只有“第三根危险上影线 + 破位确认”才触发清仓。
- 破位确认包括：关键价位突破失败、15分钟上升趋势线跌破、跌破最近一次创出新高后的回调低点、连续两根15分钟K线收在MA20下方且最后一根放量、日线放量跌破5日均线。
- 若股价仍位于日线 A/B 级强支撑区上方，第三根危险上影线触发的清仓会降级为减仓；无破位确认的减仓可继续被支撑保护降级为观察。
- 若当前价未跌破日线 A 级支撑区，且距离 A 级支撑位比距离 A 级压力位更近，则不报告减仓或清仓，继续观察。该过滤不拦截用户手动设置的硬性清仓规则和止损规则。
- 若当前价未跌破日线 B 级支撑区，且距离 B 级支撑位比距离 B 级压力位更近，则不报告减仓；但真正清仓级破位确认仍可报告。

策略会优先判断是否接近日线 A/B 级关键价位；未接近日线关键价位时，通常不会启动 15 分钟卖出监测，以减少普通 K 线造成的误报。

## 5. 回放某一天的建议

用于检查“如果回到某一天，当时策略会给出什么建议”：

```powershell
.\scripts\replay_advice.ps1 -Symbol 002241 -AsOfDate 2026-05-20
```

注意：15 分钟历史数据受在线数据源限制。如果数据源只提供最近一段时间的分钟线，太早的日期可能无法回放。

## 6. 回测一段时间

回测多只股票：

```powershell
.\scripts\backtest_strategy.ps1 -Symbols "002241,002739" -StartDate 2025-11-20 -EndDate 2026-05-27
```

在 PowerShell 中建议给多个股票代码加引号，避免逗号参数被错误解析。

### 历史15分钟数据源

工具现在会优先使用本地缓存；如果缓存数量不够，或缓存覆盖不到回测/回放日期，会继续尝试更新网络数据源。

当前网络数据源顺序：

- 历史日线、历史15分钟线：优先尝试 BaoStock。
- 若 BaoStock 不可用：回退到 AkShare/东方财富，再回退到 Sina。
- 实时行情：仍优先使用 AkShare/东方财富。

BaoStock 诊断命令：

```powershell
.\scripts\diagnose_baostock.ps1 -Symbol 002241 -StartDate 2025-01-01 -EndDate 2026-05-27
```

如果 PowerShell 阻止运行 `.ps1`，可以改用 Python 模块：

```powershell
C:\Users\admin\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe -m sell_monitor.app.diagnose_baostock --symbol 002241 --start-date 2025-01-01 --end-date 2026-05-27
```

当前回测判定口径：

- 减仓命中：触发后 15 个交易日内最大回撤 `>= 7%`。
- 清仓命中：触发后 15 个交易日内最大回撤 `>= 7%`。
- 误报：未达到对应回撤阈值，且触发后最大上涨 `>= 7%`。
- 漏报：`HOLD` 后 15 个交易日内最大回撤 `>= 7%`，且此前 15 个交易日内没有出现减仓、清仓或止损提醒。
- 已预警：`HOLD` 后 15 个交易日内最大回撤 `>= 7%`，但此前 15 个交易日内已经出现过减仓、清仓或止损提醒，因此不再计为漏报。

## 7. 常见问题

### 没有卖出提醒是不是代表工具没运行？

不一定。可能是：

- 当前价格没有接近日线 A/B 级关键价位。
- 15 分钟线没有形成合格的危险上影线。
- 分数不足 3 分，因此只给出持有或观察结论。
- 数据源临时不可用，工具回退到缓存或提示获取失败。

### 数据获取失败怎么办？

AkShare 底层会访问东方财富、Sina 等在线接口。若接口临时断开、被限流或网络不稳定，可能出现 15 分钟数据获取失败。

可尝试：

```powershell
.\scripts\diagnose_akshare_env.ps1
```

也可以稍后重试。工具已经加入本地缓存，成功拉取过的日线、15 分钟线和日线关键价位会优先复用。
