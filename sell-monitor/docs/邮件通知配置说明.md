# 邮件通知配置说明

## 1. 现在已经支持什么

邮件通知现在支持两类信号：

- 卖出监控：`reduce / stop_loss / exit_all`
- 开仓检查：`allow_entry`

不会给 `hold`、`reject_entry` 这类非动作结果发邮件，避免刷屏。

## 2. 163 邮箱推荐配置

你的收件地址已经按这个邮箱准备：

`15617592163@163.com`

如果你就用这一个 163 邮箱给自己发邮件，推荐这样配置：

```dotenv
SELL_MONITOR_SMTP_HOST=smtp.163.com
SELL_MONITOR_SMTP_PORT=465
SELL_MONITOR_SMTP_USE_SSL=true
SELL_MONITOR_SMTP_USE_TLS=false
SELL_MONITOR_SMTP_USERNAME=15617592163@163.com
SELL_MONITOR_SMTP_PASSWORD=把这里替换成163邮箱SMTP授权码
SELL_MONITOR_EMAIL_FROM=15617592163@163.com
SELL_MONITOR_EMAIL_TO=15617592163@163.com
SELL_MONITOR_EMAIL_SUBJECT_PREFIX=[SellMonitor]
```

## 3. SMTP 授权码怎么拿

常见流程是：

1. 登录 163 邮箱网页版
2. 打开邮箱设置
3. 找到 `POP3/SMTP/IMAP` 或客户端授权相关设置
4. 开启 `SMTP`
5. 获取授权码
6. 把授权码填到 `SELL_MONITOR_SMTP_PASSWORD`

注意：

- 这里填的通常不是邮箱网页登录密码
- 而是客户端专用授权码

## 5. 如何测试邮件链路

进入项目目录后运行：

```powershell
Set-Location "C:\Users\admin\Documents\New project\sell-monitor"
.\scripts\send_test_email.ps1
```

如果要自定义测试邮件标题：

```powershell
.\scripts\send_test_email.ps1 -Subject "[SellMonitor] 邮件链路测试"
```

成功时会看到：

```text
测试邮件已发送到 15617592163@163.com
```

## 6. 正式运行时怎么发

```powershell

```

开仓检查：

```powershell
.\scripts\scan_entry.ps1
```

## 7. 邮件内容规则

卖出邮件会带上：

- 股票代码或名称
- 动作
- 触发时价格
- 下一步建议
- 全部理由

开仓邮件会带上：

- 股票代码或名称
- 开仓模型
- 计划价
- 止损价
- 第一止盈位
- 盈亏比
- 全部理由

## 8. 如果测试邮件发不出去

优先检查这几项：

- `SELL_MONITOR_SMTP_PASSWORD` 是否填成了网页登录密码
- `SELL_MONITOR_SMTP_PORT` 是否和 SSL/TLS 方式匹配
- `SELL_MONITOR_SMTP_USE_SSL=true` 时，`SELL_MONITOR_SMTP_USE_TLS` 应为 `false`
- `SELL_MONITOR_EMAIL_FROM` 是否和 SMTP 账号一致
- 163 邮箱的 SMTP / 客户端授权是否真的开启

如果你之后想改成别的发件邮箱，也只要改这几项配置，不用改代码。
