## 3. 卖出主链路

卖出主链路入口在：

- `sell_monitor/app/main.py`

实际流程：

1. 读取卖出 watchlist、用户规则、配置
2. 创建 provider 与缓存回退链路
3. 如启用 Obsidian，先检查并补齐前两个交易日的缺失槽位
4. 对每只股票生成日线上下文与关键价位
5. 在激活区内汇总 15 分钟转弱信号
6. 做评分、硬规则、支撑保护、基本面权重
7. 输出到终端、Obsidian、Telegram
8. 写入复盘状态并更新回测记录

## 4. 开仓主链路

开仓链路与卖出链路平行，不混用评分语义。

入口：

- `app/entry_scan.py`
- `app/replay_entry.py`
- `app/backtest_entry.py`

当前输出三条路线：

- `standard_entry`
- `t_reentry`
- `reject_reentry`

## 5. 关键模块职责

### 5.1. `config.py`

- 读取 Obsidian 配置文件
- 解析 Markdown 包裹的 env 内容
- 提供 sell / entry 两套 watchlist 路径

### 5.2. `watchlist_factory.py`

- 根据路径后缀自动选择 JSON 或 Markdown watchlist store

### 5.3. `obsidian_watchlist_store.py`

- 支持在 `.md` 文件中维护 watchlist
- 底层仍按 JSON 结构读写

### 5.4. `monitor/obsidian_backfill.py`

- 检查固定时间槽位是否缺失
- 按当时数据回溯补齐

### 5.5. `notifier/channels/obsidian.py`

卖出 Obsidian 输出：

- 单股文档
- 顶部最新支撑压力图
- 唯一结果表
- 当日应卖出汇总
- 复盘状态回写

### 5.6. `notifier/channels/entry_obsidian.py`

开仓 Obsidian 输出：

- 单股开仓文档
- 顶部最新支撑压力图
- 开仓路线/模型表
- 当日可开仓候选汇总

### 5.7. `notifier/channels/telegram.py`

- 负责 Telegram 发送
- 卖出和开仓主程序都复用它

## 6. 文档与配置的当前事实

当前不是单一 `watchlist.json` 模式，而是：

- 卖出：`sell-watchlist.md`
- 开仓：`entry-watchlist.md`

当前不是所有输出都写到 `monitor/` 根目录，而是：

- 卖出写到 `monitor/sell/`
- 开仓写到 `monitor/entry/`

当前卖出汇总文件名不是 `当日触发.md`，而是：

- `当日应卖出.md`
