# 卖出提醒工具开发进展

更新时间：2026-06-25

## 1. 当前定位

- 范围：仅 A 股
- 卖出侧：实时监控、回放、回测
- 开仓侧：实时检查、回放、回测
- 通知：终端、Obsidian、Telegram
- 不做自动下单

## 2. 已完成

### 2.1. 卖出主流程

- [x] 实时卖出监控
- [x] 仅在 A 股交易时间实时运行
- [x] 支持 watchlist 全量和单股运行
- [x] 指定股票时自动加入卖出 watchlist

### 2.2. 数据源与缓存

- [x] AkShare 接入
- [x] BaoStock 历史数据接入
- [x] Hybrid provider
- [x] 缓存回退
- [x] 日线缓存
- [x] 15 分钟缓存
- [x] quote 缓存
- [x] 关键价位缓存
- [x] 基本面缓存

### 2.3. 卖出策略

- [x] 日线关键价位分级
- [x] 周线压力位纳入
- [x] 日线斐波那契压力位纳入
- [x] 危险上影线三段式识别
- [x] 突破失败识别
- [x] 放量滞涨 / 量价异常识别
- [x] 趋势线跌破识别
- [x] 结构低点破位识别
- [x] gap risk

## 3. 当前已经修正的旧说法

- [x] 实时监控不再依赖 `positions`
- [x] 不再使用单一 `watchlist.json` 作为主配置
- [x] 卖出汇总文件不再叫 `当日触发.md`
- [x] 输出目录不是 `monitor/` 根目录，而是分为 `sell/` 与 `entry/`
- [x] 开仓结果现在有三条路线，不再只有允许/不允许

## 4. 仍需注意的限制

- AkShare 和东方财富接口稳定性仍受外部环境影响
- 历史分钟线覆盖范围受在线源和本地缓存共同限制
- 60 分钟确认目前仍主要通过保守代理结构实现，不是完整独立分钟数据链路

## 5. 当前结论

这套系统现在已经不是原型，而是一套能日常使用的 A 股卖出监控 + 开仓检查工具。
当前最大的外部不确定性仍然是数据源稳定性，而不是主规则链路本身。
