# 卖出提醒工具使用说明

本文只描述当前代码已经实现的行为。

## 1. 配置文件与默认路径

默认读取的 Obsidian 配置文件：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\sell-monitor-config.md
```

当前模板里的主要配置：

```dotenv
SELL_MONITOR_PROVIDER=akshare
SELL_MONITOR_OBSIDIAN_MONITOR_DIR=E:\tools\OB\Obsidian\Trade\notes\monitor\sell
SELL_MONITOR_OBSIDIAN_ENTRY_DIR=E:\tools\OB\Obsidian\Trade\notes\monitor\entry
SELL_MONITOR_SELL_WATCHLIST_PATH=E:\tools\OB\Obsidian\Trade\notes\monitor\config\sell-watchlist.md
SELL_MONITOR_ENTRY_WATCHLIST_PATH=E:\tools\OB\Obsidian\Trade\notes\monitor\config\entry-watchlist.md
SELL_MONITOR_POSITIONS_PATH=E:\tools\OB\Obsidian\Trade\notes\monitor\config\positions.md
SELL_MONITOR_USER_RULES_PATH=E:\tools\OB\Obsidian\Trade\notes\monitor\config\user_rules.md
```

说明：

- 卖出监控输出目录是 `monitor/sell`
- 开仓检查输出目录是 `monitor/entry`
- 若找不到配置文件，代码层的 provider 回退默认值是 `static`

## 2. watchlist 的实际位置与写法

卖出监控只读取：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\sell-watchlist.md
```

开仓检查只读取：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\entry-watchlist.md
```

这两个文件是 Markdown，但实际内容读取的是第一个代码块里的 JSON。

示例：

```json
{
  "symbols": [
    {
      "symbol": "002241",
      "name": "歌尔股份"
    },
    {
      "symbol": "300014",
      "name": "亿纬锂能"
    },
    "601888"
  ]
}
```

规则：

- `symbol` 必须是 6 位 A 股代码
- `name` 可选；填了以后，Obsidian 表格里的“股票代码”列优先显示中文名
- 不填 `name` 时，显示股票代码
- 命令行带 `--symbol` 运行时，如果股票不在对应 watchlist 中，会自动加入该 watchlist

## 3. positions 与 user rules

当前实时卖出监控不会因为没有 `positions.md` 而跳过。

也就是说：

- 没有持仓文件，实时监控照常运行
- 不再出现“未持仓所以不检测”的逻辑
- `positions.md` 仍会被读取，主要给回溯补齐和兼容逻辑使用

用户规则文件：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\user_rules.md
```

用于保存：

- 预设止损位
- 预设止盈位
- 硬性清仓条件
- 原始交易理由

## 4. 启动命令

进入项目目录：

```powershell
Set-Location "C:\Users\admin\Documents\New project\sell-monitor"
```

单次监控全部卖出 watchlist：

```powershell
.\scripts\run_akshare.ps1
```

单次监控单只股票：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241
```

每 1 小时循环监控：

```powershell
.\scripts\run_akshare.ps1 -Loop -IntervalSeconds 3600
```

非交易时段强制运行一次：

```powershell
.\scripts\run_akshare.ps1 -Symbol 002241 -IgnoreTradingHours
```

## 5. 交易时间限制

实时卖出监控默认只在 A 股常规交易时段运行：

- `09:30-11:30`
- `13:00-15:00`

非交易时段会直接跳过本轮实时检测。

## 6. 数据源与缓存

当前支持的 provider 链路：

- `akshare`
- `baostock`
- `static`
- `miniqmt`

常用行为：

- `akshare` 模式下，实时行情优先 AkShare，历史数据可混合 BaoStock，再套本地缓存回退
- `baostock` 模式下，历史数据优先 BaoStock，实时价仍尽量补 AkShare
- `static` 模式下，使用 `examples/market_data.json`

本地缓存内容：

- quote
- 日线
- 15 分钟线
- 日线关键价位
- 基本面快照

当在线数据失败时，会优先回退本地缓存。

## 7. 卖出监控结果写到哪里

### 7.1. 单股文档

每个股票一篇文档，位于：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\sell\002241.md
```

文档结构固定为：

1. 标题
2. 最新支撑压力图
3. 唯一一张监控结果表

表格列为：

- `检测时间`
- `股票代码`
- `结论`
- `动作`
- `分数`
- `价格`
- `原因/提示`
- `下一步`
- `取消条件`
- `复盘`

说明：

- “股票代码”列如果 watchlist 配了 `name`，优先显示中文名
- `价格` 列显示报告时刻的价格，不是开盘价/收盘价
- 卖出类动作 `reduce / stop_loss / exit_all` 的时间会标红
- 即使没有卖出信号，也会写一条记录

### 7.2. 当日汇总

当日触发卖出类动作的股票，会汇总到：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\sell\当日应卖出.md
```

表格列为：

- `时间`
- `股票代码`
- `卖出动作`
- `分数`
- `价格`
- `原因`
- `建议`
- `复盘`

只有 `reduce / stop_loss / exit_all` 会进入这份汇总。

### 7.3. 支撑压力图

单股文档顶部只保留一张最新支撑压力图：

- 周线 K 线背景
- 绿色透明矩形代表支撑
- 红色透明矩形代表压力
- 等级越高，颜色越深

## 8. 自动回溯补齐

如果实时监控运行前，Obsidian 卖出文档中缺少前两个交易日到当前时刻的固定槽位记录，系统会自动补齐。

检查槽位：

- `09:30`
- `10:30`
- `11:30`
- `13:00`
- `14:00`
- `15:00`

补齐规则：

- 只用该时刻及之前的数据重算
- 补齐结果写回对应单股文档
- 若补齐过程中出现卖出信号，Telegram 会按股票汇总发送“回溯补齐卖出汇总”

## 9. 回放与回测

单日回放：

```powershell
.\scripts\replay_advice.ps1 -Symbol 002241 -AsOfDate 2025-10-09
```

区间回测：

```powershell
.\scripts\backtest_strategy.ps1 -Symbols "002241,002739" -StartDate 2025-11-20 -EndDate 2026-05-27
```

注意：

- PowerShell 里多个股票代码要作为一个字符串传入
- 不要写成 `-Symbols 002241,002739`

默认回测输出目录：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\sell\backtest
```

## 10. 当前回测判定口径

- 减仓命中：触发后 15 个交易日内最大回撤 `>= 7%`
- 清仓命中：触发后 15 个交易日内最大回撤 `>= 7%`
- 误报：未达到对应回撤阈值，且触发后最大上涨 `>= 7%`
- 漏报：`hold` 后 15 个交易日内最大回撤 `>= 7%`，且此前 15 个交易日内没有出现减仓、清仓或止损提醒

## 11. 常见问题

### 11.1. 为什么没有卖出提醒

常见原因：

- 当前价格没有进入日线关键价位激活区
- 15 分钟没有形成合格转弱结构
- 支撑保护或支撑偏置过滤把普通卖出信号压回观察
- 数据源临时不可用，只留下了提示信息

### 11.2. 为什么总是数据获取失败

可能原因：

- AkShare 依赖的东方财富接口临时不可用
- BaoStock 当前环境不可用
- 网络访问被拦截
- 本地缓存也不够

建议优先诊断：

```powershell
.\scripts\diagnose_akshare_env.ps1
.\scripts\diagnose_baostock.ps1 -Symbol 002241 -StartDate 2025-01-01 -EndDate 2026-05-27
```

### 11.3. 为什么表格里看到的是中文名

因为当前显示逻辑是：

- 若 watchlist 配置了 `name`，优先显示中文名
- 若未配置，回退显示股票代码
