# 监控中心使用说明

## 目录结构

建议将所有相关文件统一放在：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor
```

推荐结构：

```text
monitor/
  config/
    sell-monitor-config.md
    sell-watchlist.md
    entry-watchlist.md
    positions.md
    user_rules.md
  sell/
  entry/
  launchers/
  docs/
```

## 配置文件

主配置文件：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\sell-monitor-config.md
```

卖出监控名单：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\sell-watchlist.md
```

开仓检查名单：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\config\entry-watchlist.md
```

说明：

- 卖出监控只读取 `sell-watchlist.md`
- 开仓检查只读取 `entry-watchlist.md`
- 两份名单互不影响，可以分别维护
- 如果某只股票同时要做卖出监控和开仓检查，就在两份名单里都加上

## Watchlist 写法

示例：

```json
{
  "symbols": [
    {
      "symbol": "002241",
      "name": "歌尔股份"
    },
    {
      "symbol": "300015",
      "name": "爱尔眼科"
    },
    "300014"
  ]
}
```

说明：

- `symbol` 是股票代码
- `name` 可选，填了就优先显示中文名
- 不写 `name` 时，系统回退显示股票代码

## 一键启动

把启动器放到：

```text
E:\tools\OB\Obsidian\Trade\notes\monitor\launchers
```

常用启动器：

- `启动卖出监控.bat`
- `启动卖出监控_每小时循环.bat`
- `启动开仓检查.bat`
- `启动开仓检查_每小时循环.bat`
- `回放卖出建议_示例.bat`
- `回放开仓建议_示例.bat`
- `策略回测_示例.bat`
- `开仓回测_示例.bat`

如果 Obsidian 不显示 `.bat`，同目录可以配一份 `.md` 启动说明笔记。

## Shell Commands 插件

你已经装了 `Shell commands` 插件，推荐这样配：

1. 新建命令
2. Shell 选择 `PowerShell`
3. 命令内容直接指向对应启动器
4. 给常用命令绑定快捷键或加入命令面板

PowerShell 示例：

```powershell
& "E:\tools\OB\Obsidian\Trade\notes\monitor\launchers\启动卖出监控.bat"
```

```powershell
& "E:\tools\OB\Obsidian\Trade\notes\monitor\launchers\启动开仓检查.bat"
```

如果运行没反应，优先检查：

- Shell 是否选成了 `PowerShell`
- 路径是否和本机实际路径一致
- 插件是否允许在后台运行命令
- 启动器里调用的 Python 环境是否存在

## Telegram 提醒

如果要用 Telegram 提醒，在 `sell-monitor-config.md` 里补这几项：

```dotenv
SELL_MONITOR_TELEGRAM_BOT_TOKEN=your-bot-token
SELL_MONITOR_TELEGRAM_CHAT_ID=your-chat-id
SELL_MONITOR_TELEGRAM_SUBJECT_PREFIX=[SellMonitor]
```

触发条件：

- 卖出监控：卖出分数 `>= 8`，且动作是 `reduce`、`stop_loss`、`exit_all`
- 开仓检查：`allowed=yes`

详细步骤见 `Telegram通知使用说明.md`。

## 输出位置

- 卖出监控结果写入 `sell/`
- 开仓检查结果写入 `entry/`
- 配置统一放在 `config/`
- 使用说明和操作文档统一放在 `docs/`
